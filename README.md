# RAG Chatbot

A small but complete Retrieval-Augmented Generation chatbot. Teach it text, then ask questions вЂ” it answers **only** from what you've taught it.

```
User Query в†’ Embedding в†’ Vector Search в†’ Context в†’ Gemini LLM в†’ Grounded Answer
```

- **Backend** вЂ” Python + FastAPI, Gemini API for embeddings & generation, file-backed vector store with cosine similarity (no DB to install).
- **Frontend** вЂ” React + Vite, two-tab UI (Chat / Teach).
- **Zero hallucinations** вЂ” strict prompt + similarity threshold; off-topic questions get `"I don't know"`.

---

## 1. Prerequisites

- **Python 3.9+**
- **Node.js 18+** (for the frontend only)
- A free Gemini API key вЂ” get one at <https://aistudio.google.com/app/apikey>

---

## 2. Backend Setup (FastAPI)

```bash
cd backend
cp .env.example .env
# open .env and paste your GEMINI_API_KEY

python3 -m venv .venv
.venv/bin/pip install -r requirements.txt

.venv/bin/uvicorn main:app --reload --port 8000
```

The server starts on <http://localhost:8000>. Quick health check:

```bash
curl http://localhost:8000/health
# в†’ {"ok":true,"service":"rag-chatbot-backend"}
```

Interactive API docs (auto-generated by FastAPI) live at <http://localhost:8000/docs>.

### Backend endpoints

| Method | Path           | Body                                   | Purpose                                |
| ------ | -------------- | -------------------------------------- | -------------------------------------- |
| POST   | `/train`       | `{ "content": "..." }`                 | Chunk + embed + store the text         |
| DELETE | `/train/clear` | вЂ”                                      | Wipe the knowledge base                |
| GET    | `/train/stats` | вЂ”                                      | Returns `{ totalChunks }`              |
| POST   | `/chat`        | `{ "message": "...", "history": [] }`  | RAG query в†’ answer + sources           |

## 6. How grounding works

Two safeguards prevent hallucinations:

1. **Similarity floor** (`MIN_SCORE = 0.35` in `backend/routes/chat.py`). If no chunk crosses the threshold, the backend short-circuits and returns `"I don't know"` *without* even calling the LLM.
2. **Strict prompt** in `backend/routes/chat.py`:
   ```
   You are a helpful assistant. Answer ONLY using the context below.
   If the answer is not present in the context, reply exactly: "I don't know".
   Do not invent facts. Be concise and direct.
   ```

Tune `MIN_SCORE` and `TOP_K` at the top of `backend/routes/chat.py` to trade recall vs. precision.

## 7. Models

Defaults (set in `backend/.env.example`):

- **Embeddings** вЂ” `gemini-embedding-001`
- **Generation** вЂ” `gemini-2.5-flash-lite`

Override either via `GEMINI_EMBED_MODEL` / `GEMINI_CHAT_MODEL` in `.env`. If you hit free-tier rate limits, the `*-lite` variants generally have the most generous quotas.

To list every model your key can access:

```bash
curl -s "https://generativelanguage.googleapis.com/v1beta/models?key=$GEMINI_API_KEY" \
  | python3 -c "import json,sys; [print(m['name'], '-', ','.join(m.get('supportedGenerationMethods',[]))) for m in json.load(sys.stdin)['models']]"
```

## 8. Swapping the vector store

`backend/vector_db/store.py` exposes a tiny interface вЂ” `add`, `search`, `count`, `clear`. To swap in FAISS, ChromaDB, pgvector, etc., reimplement those four async functions and the rest of the system stays the same.

---

## 9. Notes

- The store file is `backend/data/store.json`. Delete it (or call `DELETE /train/clear`) to start fresh.
- Embeddings are generated sequentially to stay friendly to free-tier rate limits. Batch in parallel if you have a paid key.
- The `.env` file is gitignored вЂ” never commit your API key.
